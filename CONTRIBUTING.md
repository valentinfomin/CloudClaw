# Contributing to CloudClaw

First off, thank you for considering contributing to CloudClaw! It's people like you that make such a project thrive.

## How Can I Contribute?

### Reporting Bugs
If you come across a bug, please open a new issue. To help us resolve it quickly, please include:
- A clear and descriptive title.
- A detailed description of the problem, including steps to reproduce it.
- Your environment details (e.g., Node.js version, OS).
- Any relevant logs or screenshots.

### Suggesting Enhancements
If you have an idea for a new feature or an improvement to an existing one, we'd love to hear about it. Please open an issue with a clear title and a detailed description of your suggestion.

### Pull Requests
We welcome pull requests! Please follow these steps:
1. Fork the repository and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure your code lints and follows the existing style.
4. Issue that pull request!

## Styleguides
We use a standard JavaScript style. Please ensure your contributions adhere to the conventions you see in the existing codebase.

## Code of Conduct
This project and everyone participating in it is governed by the [CloudClaw Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.
